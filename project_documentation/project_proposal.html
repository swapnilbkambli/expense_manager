Project Proposal: Personal Expense Analytics Dashboard

A high-performance, client-side reporting tool built with Next.js to transform raw CSV expense data into actionable financial insights.

1. Core Tech Stack

Next.js 14 (App Router) Tailwind CSS Shadcn UI Recharts PapaParse (CSV Engine) date-fns Lucide React Icons

2. Key Features

  • Dynamic File Ingestion: A drag-and-drop CSV uploader that parses files locally. No data leaves the browser, ensuring 100% privacy.
  • Advanced Multi-Filtering: Global filters for Category and Subcategory using multi-select searchable dropdowns. These filters dynamically update based on the uploaded file's unique values.
  • Temporal Intelligence: Pre-defined date ranges (YTD, All Time, Last 30 Days) and a Custom Calendar Date Range picker.
  • Interactive Visualizations: Real-time refreshing charts including:
    • Trend Line: Daily/Monthly spending over time.
    • Composition Donut: Percentage share of spending by Category.
    • Subcategory Bar Chart: Ranking of highest spending areas.
  • Summary Metrics: Top-level "Stat Cards" showing Total Income, Total Expenses, and Net Savings for the selected period.

3. Data Structure Mapping

The application will map your specific CSV headers to the UI logic:

  • Date (Format: DD-MM-YYYY) -> Temporal filtering
  • Amount (Negative for expenses, Positive for income) -> Metric calculation
  • Category -> Primary grouping
  • Subcategory -> Secondary drill-down

4. Proposed UI Layout

Header: App title and "Upload New CSV" button.
Visual: Clean, sticky navigation with a dark/light mode toggle.
Filter Bar: Horizontal bar with Date Range Picker, Category Multi-select, and Subcategory Multi-select.
Visual: Flexible grid that wraps on mobile. Categories show counts (e.g., "Food (12)").
Dashboard Grid: - Row 1: Three Stat Cards (Income, Expense, Savings). - Row 2: Large Line Chart (Monthly Trends). - Row 3: Donut Chart (Category Split) & Vertical Bar Chart (Subcategory breakdown).
Transaction Table: A searchable, sortable list of all transactions matching current filters.
Visual: Styled with Shadcn Data Tables with pagination.

5. Implementation Plan

  1. Phase 1: Setup Next.js environment and Shadcn UI components.
  2. Phase 2: Build the CSV parsing engine using PapaParse with date formatting logic.
  3. Phase 3: Implement the global state for filters (Date/Category/Subcategory).
  4. Phase 4: Integrate Recharts with responsive containers that listen to filter changes.
  5. Phase 5: Final Polish: Add "Empty State" illustrations and loading skeletons.
